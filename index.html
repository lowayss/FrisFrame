FrisFrame GUIDE

기능 매뉴얼

스토리보드부터 카메라 영상까지, 현재 구현된 작업 흐름과 규칙을 확인합니다.

01

전체 작업 흐름

하나의 컷을 설계하고 결과를 확인하는 기본 순서입니다.

0초
5초
3D 카메라 프레임
영화적 헤드룸
0초5초
키 두 개가 있어야 이동합니다
따라 하기 · 5초 이동 숏

배우가 카메라 쪽으로 걸어오는 컷

  1. 0초 배우와 카메라의 시작 위치를 잡고 첫 키를 추가합니다.
  2. 5초 배우를 카메라 쪽으로 옮긴 뒤 두 번째 키를 추가합니다.
  3. 추적 카메라의 피사체 추적을 배우로 설정해 얼굴을 유지합니다.
  4. 검토 재생하고 3D 카메라 프레임에서 헤드룸을 확인합니다.
2D 위치, 시간, 카메라 구도가 하나의 컷으로 연결됩니다.
  1. 프로젝트 구조시나리오·콘티에서 씬과 컷을 만들거나 스토리보드에서 직접 추가합니다.
  2. 2D 배치배우, 소품, 차량과 카메라를 평면도에서 배치하고 방향을 정합니다.
  3. 카메라 확정렌즈, 높이, 팬·틸트와 피사체 추적으로 영화적 구도를 잡습니다.
  4. 동선 기록시간을 옮겨 키프레임을 추가하고 경로와 도착 방식을 정합니다.
  5. 3D 검토3D 무대와 카메라 프레임에서 거리, 높이, 헤드룸과 움직임을 확인합니다.
  6. 프리뷰 후 저장현재 프레임, 시작·끝 프레임 또는 MP4를 먼저 확인한 뒤 직접 저장합니다.

튜토리얼과 설명서의 차이 상단 `도움말`의 처음 사용 가이드는 화면을 따라가는 안내이고, 기능 매뉴얼은 필요한 내용을 검색해 확인하는 참고 문서입니다.

02

프로젝트와 시나리오·콘티

여러 씬과 컷을 하나의 프로젝트 문서로 관리합니다.

프로젝트 저장계속 편집할 작업본내 프로젝트 목록과 자동저장
JSON 백업옮기거나 보관할 사본프리뷰 뒤 직접 파일 저장
공유 링크다른 사람에게 보여줄 사본원본 편집과 따로 유지

프로젝트 시작과 목록

  • 처음 시작할 때는 프로젝트 제목을 정해 새로 만들거나 기존 프로젝트를 먼저 엽니다.
  • 프로젝트 저장: 상단 `프로젝트` 메뉴에서 현재 작업을 즉시 저장합니다.
  • 프로젝트 관리: 최근 작업을 검색하고 열거나 이름 변경, 복제할 수 있습니다.
  • 새 프로젝트: 제목과 서버 저장이 완료된 뒤 기본 컷으로 시작합니다.
  • 한 번 만든 프로젝트는 편집 후 자동 저장되며 다음 실행 때 마지막 프로젝트를 이어서 엽니다.

시나리오 구조 분석

  • `INT./EXT.`, 장면 제목과 `컷`, `액션`, `대사`, `렌즈`, `길이` 표기를 읽습니다.
  • 적용 전 씬·컷 수와 경고를 검토할 수 있습니다.
  • `전체 시나리오 구조 교체`는 기존 씬 구조를 바꾸고, `뒤에 추가`는 기존 프로젝트 다음에 붙입니다.

백업과 웹 공유

  • JSON 백업 준비는 서버 저장과 별개로 이동용 파일을 프리뷰합니다.
  • 공유할 때 현재 프로젝트의 별도 사본과 링크가 생성됩니다.
  • 공유 후 수정한 내용은 기존 링크에 자동 반영되지 않습니다.
  • JSON과 공유 링크 모두 자동으로 파일을 내려받지 않습니다.

저장 안전장치

  • `변경됨`, `저장 중`, `저장됨`, `저장 충돌`, `저장 실패`로 상태를 구분합니다.
  • 다른 창에서 먼저 저장되면 자동 덮어쓰기 대신 충돌을 표시하고 현재 편집본을 새 프로젝트로 보존할 수 있습니다.
  • 삭제는 휴지통을 거치며 복원할 수 있습니다. 영구 삭제는 휴지통 안에서만 가능합니다.
  • 실행 취소 기록과 생성된 대표 프레임 이미지는 프로젝트 문서에 포함되지 않습니다.

프로젝트 교체 새 프로젝트, JSON 불러오기와 `전체 시나리오 구조 교체`는 현재 프로젝트 전체를 바꿀 수 있습니다. 저장하지 않은 변경과 확인 창의 씬·컷 영향을 읽고 진행하세요.

03

스토리보드

컷 순서, 내용, 진행 상태와 연속성을 한 화면에서 검토합니다.

씬과 컷

  • 왼쪽에서 씬을 선택하고 `+`로 씬을 추가합니다.
  • 중앙의 컷 카드는 대표 카메라 프레임, 샷 크기, 렌즈, 길이와 키 개수를 보여줍니다.
  • 컷을 두 번 클릭하거나 `블로킹 열기`를 누르면 해당 컷의 2D·3D 작업으로 이동합니다.

컷 정보

  • 오른쪽에서 상태, 제목, 샷 크기, 액션, 대사, 연출 의도와 연속성 메모를 편집합니다.
  • 상태는 `초안 → 블로킹 중 → 검토 필요 → 확정`으로 구분합니다.
  • `확인` 배지는 키 누락과 배우·소품 연속성 변화를 알려줍니다.

보기 범위

  • 스토리보드는 항상 전체 시나리오의 모든 씬과 컷을 순서대로 표시합니다.
  • 왼쪽 씬 목록을 누르면 전체 화면 안에서 해당 씬으로 이동합니다.
  • 상태 필터로 초안이나 검토 대상만 모아 볼 수 있습니다.

카메라 자동 가배치

  • 콘티/시나리오 텍스트를 불러오면 본문 속 카메라 키워드를 분석해 3D 씬을 자동으로 배치합니다.
  • `ECU`/`ELS` 등의 지시어로 카메라 거리와 화각(`focal`)을 결정합니다.
  • `정면`/`측면`/`후면` 및 `하이`/`로우`/`수직` 앵글 단어를 파싱해 카메라 좌표와 높이, 틸트를 알아서 조절합니다.

이미지 묶기

  • 전체 시나리오와 상태 필터를 기준으로 4컷, 6컷, 12컷 시트를 만듭니다.
  • 한 페이지면 PNG, 여러 페이지면 PNG 묶음 ZIP으로 준비됩니다.
04

2D 무대와 대상 배치

탑뷰 평면도에서 카메라, 배우와 소품의 공간 관계를 설계합니다.

대상을 드래그배우·소품·카메라 이동
방향 지정화살표가 바라보는 곳
전체 맞춤롱숏 동선을 한눈에 확인

무대

  • 16:9, 9:16, 4:3, 1:1, 3:4 화면비를 선택합니다.
  • 무대의 긴 변은 36m이며 16:9 기준 약 36×20.3m입니다.
  • 무대는 작업창보다 작아지지 않으며 100%에서 전체 영역이 보입니다.
  • 휠로 최대 400%까지 확대하고, 확대된 빈 무대를 좌클릭 드래그해 보이는 영역을 이동합니다.
  • 가운데 버튼 또는 `Space + 드래그`는 대상 위에서도 무대를 이동합니다.
  • 오른쪽 위 맞춤 버튼은 무대 전체가 보이는 100% 배율로 돌아갑니다.
  • 격자와 이름 표시, 카메라 시야 표시를 켜고 끌 수 있습니다.
  • 공간 프리셋은 거실, 주방, 침실, 숲, 차 안의 기본 배치를 추가합니다.
  • `무대 리셋`은 현재 컷의 배치와 키프레임만 기본값으로 되돌립니다.

배우·소품 추가

  • 이름을 입력해 배우를 추가하고, 소품은 유형을 선택한 뒤 추가합니다.
  • 캔버스에서 대상을 드래그해 위치를 옮깁니다.
  • 속성에서 이름, 색, 크기, 방향과 소품의 길이·높이·너비를 조정합니다.

방향과 동선 안내

  • 방향 슬라이더나 8방향 버튼으로 배우와 소품이 바라보는 방향을 정합니다.
  • 2D에서 `Shift`를 누른 채 드래그하면 이동과 함께 방향을 맞출 수 있습니다.
  • 동선은 파랑(카메라), 청록(배우), 황색(소품)으로 구분하며 경로의 화살표가 진행 방향을 표시합니다.
  • 현재 구간에는 선택한 도착 키의 지문이 화살표 옆에 표시됩니다.
  • 빈 원은 시작, 밝은 점은 현재 위치, 십자 표적은 다음 도착점을 뜻합니다.
  • 미세 이동 버튼과 방향키는 작은 위치 보정에 사용합니다.

카메라 표시

  • 카메라 본체를 드래그하면 위치가 이동합니다.
  • 카메라의 시야 영역은 현재 렌즈와 회전 방향을 반영합니다.
  • 포커스 표시는 카메라가 향하는 지점을 뜻합니다.

선택과 동선 대상 캔버스에서 대상을 선택하면 타임라인의 활성 대상도 함께 바뀝니다.

05

3D 프리비즈

높이와 거리감을 포함한 무대 배치와 실제 카메라 프레임을 함께 검토합니다.

대상 편집

  • G 이동: 배우, 소품과 카메라를 바닥면에서 이동합니다.
  • R 회전: 선택 대상을 회전하고 카메라는 팬 방향을 바꿉니다.
  • P 포즈: 배우 관절 구체나 팔·다리·머리 등 **신체 부위를 직접 드래그**하여 관절인형처럼 직관적으로 자세를 만듭니다. 관절을 선택하면 나타나는 **3차원 회전 기즈모 고리(빨강: X축 앞뒤, 초록: Y축 회전, 파랑: Z축 기울기)**를 잡고 드래그하면 단축키 없이 편리하게 해당 축으로만 정밀 포징이 가능합니다.

간편 포즈 시스템

  • 30+ 프리셋: 기본, 이동, 앉기/눕기, 액션, 제스처 5개 카테고리 탭 제공.
  • 복사 & 붙여넣기: 현재 배우의 포즈를 복사해 다른 배우에게 그대로 붙여넣습니다.
  • ⭐ 내 포즈: 마음에 드는 커스텀 포즈에 이름을 붙여 보관하고 필요할 때 재활용합니다.
  • 🎲 랜덤: 클릭 시 관절 각도를 난수화해 다양하고 러프한 자세를 빠르게 잡아줍니다.

화면 읽기

  • 배우, 소품, 카메라와 동선은 서로 다른 색으로 구분됩니다.
  • 마네킹의 눈·코·입과 머리 방향으로 정면과 시선을 확인합니다.
  • 오른쪽 아래 카메라 프레임은 표시값 없이 최종 구도만 보여줍니다.

항상 가능한 시점 조작

  • 좌클릭: 대상 위에서는 선택·편집하고, 무대 바닥이나 빈 공간에서는 오빗합니다.
  • 가운데 버튼도 오빗으로 동작합니다.
  • 우클릭 드래그 또는 Shift + 가운데 버튼으로 무대 전체를 화면의 좌우·위아래로 이동합니다.
  • 휠로 확대·축소하며, Ctrl + 가운데 버튼으로도 줌할 수 있습니다.

확인할 것

  • 배우가 차량이나 소품과 실제로 겹치지 않는지 확인합니다.
  • 카메라 높이 변화와 지미집·드론 경로는 측면 각도에서도 확인합니다.
06

카메라와 구도

렌즈와 카메라 자세를 정하고 얼굴·머리 기준의 영화적 구도를 만듭니다.

피사체 추적은 배우의 바닥이 아니라 얼굴·머리를 향하고, 프레임 안에서는 과한 헤드룸을 줄입니다.

렌즈

  • 14–135mm 범위에서 직접 입력하거나 14, 24, 35, 50, 85, 135mm 프리셋을 선택합니다.
  • 낮은 수치는 넓은 화각, 높은 수치는 좁은 화각과 압축된 공간감을 만듭니다.

높이·회전

  • 카메라 높이는 0.4–3.0m 범위입니다.
  • 팬은 좌우 회전, 틸트는 위아래 각도입니다.
  • `카메라 높이 키`는 같은 시간의 카메라 키가 있으면 높이만 갱신하고, 없으면 새 카메라 키를 만듭니다.

피사체 추적

  • 배우나 소품을 선택하면 카메라 팬·틸트가 대상을 계속 향합니다.
  • 배우는 발이 아니라 얼굴·머리 높이를 기준으로 계산합니다.
  • 자동 구도는 과도한 헤드룸을 줄이는 영화적 화면 위치를 사용합니다.

카메라 잠금

  • 위치, 회전, 렌즈, 높이를 각각 잠글 수 있습니다.
  • 잠긴 값은 2D·3D 드래그와 키 갱신에서도 유지됩니다.
  • 회전을 잠그면 피사체 추적이 해제됩니다.

추적과 수동 회전 피사체 추적 중에는 팬·틸트가 자동 계산되므로 수동 입력이 비활성화됩니다.

07

키프레임과 타임라인

대상별 위치·방향과 카메라 값을 시간에 따라 기록합니다.

10초23초37초
점을 좌우로 끌면 키의 시간이 바뀝니다. 점을 선택하면 아래의 도착 방식과 경로가 그 키에 적용됩니다.

키 만들기

  • `대상`에서 카메라, 배우, 소품 또는 전체를 선택합니다.
  • `현재 시간`을 정하고 `키프레임 추가`를 누르면 현재 배치가 새 키로 저장됩니다.
  • `선택 키 갱신`은 선택 키를 현재 배치와 시간으로 바꿉니다.
  • 키가 없는 대상은 전체 시간 동안 현재 위치에 고정됩니다.

시간 이동

  • 타임라인 점을 좌우로 드래그해 키 시간을 자유롭게 바꿉니다.
  • `통합`은 모든 키를 한 줄에, `대상별`은 소스마다 별도 트랙으로 표시합니다.
  • 전체 시간을 바꾸면 키 시간이 같은 비율로 재배치됩니다.

재생 제어

  • 재생: 현재 타임라인 위치부터 이어서 재생합니다.
  • 일시정지: 현재 프레임과 시간을 그대로 유지합니다.
  • 정지: 재생을 끝내고 0초의 시작 프레임으로 돌아갑니다.
  • `Space`는 재생과 일시정지를 전환합니다.

도착 방식

  • 부드럽게: 출발·도착 속도를 완화합니다.
  • 일정 속도: 구간을 같은 속도로 이동합니다.
  • 직전 유지: 도착 순간까지 이전 값을 유지합니다.
  • 즉시 전환: 해당 시간에 위치와 값을 바로 바꿉니다.

공간 경로

  • 직선, 수평·수직 직선, 좌우 원호와 자유 곡선을 선택할 수 있습니다.
  • 카메라는 드론 비행과 지미집 상승·하강 경로도 사용할 수 있습니다.
  • 자유 곡선은 2D 캔버스의 곡선 손잡이로 모양을 조정합니다.

화면 지문

  • 키를 선택하고 `지문`에 `웃으며 오른쪽으로 두 걸음`처럼 짧은 행동 설명을 적습니다.
  • 지문은 해당 키로 이동하는 구간의 화살표 옆에 표시됩니다.
  • 비워 두면 위치 변화에 맞는 기본 지문을 자동으로 만듭니다.

트랙 숨기기 대상별 타임라인의 `×`는 표시만 감추는 것이 아니라 해당 대상의 키를 삭제하고 고정 상태로 바꿉니다. 잠긴 대상에서는 실행할 수 없습니다.

소품의 동선 타임라인 소품 속성에서 이 옵션을 끄면 독립 타임라인에서 제외되고 기존 소품 키도 삭제됩니다.

잠금된 키 잠긴 대상의 키는 이동·갱신·삭제할 수 없고, 잠긴 키가 있으면 전체 시간 변경도 잠시 비활성화됩니다.

08

차량 탑승과 대상 묶음

배우와 차량·소품을 함께 움직이도록 관계를 설정합니다.

자동 탑승

  • 배우를 선택하고 `탑승 방식 → 자동 탑승`을 선택합니다.
  • 차량과 빈 좌석을 정하면 배우 위치와 방향이 차량 좌석에 자동으로 맞춰집니다.
  • 차량 키프레임을 움직이면 탑승 배우도 함께 이동합니다.

수동 탑승

  • `수동 탑승`에서는 배우를 차량 위에 직접 배치하고 방향을 맞춥니다.
  • `겹친 대상 묶기`를 사용하면 차량과 배우를 하나의 변환 묶음으로 움직일 수 있습니다.
  • `해제`하면 현재 보이는 위치를 유지한 채 각 대상을 다시 분리합니다.

좌석 규칙

  • 자동차·버스·오토바이 등 차량 유형에 정의된 좌석만 선택할 수 있습니다.
  • 이미 사용 중인 좌석은 다른 배우에게 선택되지 않습니다.
  • 차량을 일반 소품으로 바꾸거나 삭제하면 자동 탑승 관계가 해제됩니다.

묶음과 잠금

  • 묶음 구성원이나 자동 탑승 배우 중 하나가 잠기면 연결된 변환 전체가 보호됩니다.
  • `대상 편집 잠금`을 해제하면 연결된 대상도 함께 편집 가능 상태로 바뀝니다.

탑승 방식 전환 자동 탑승과 수동 탑승을 바꾸면 해당 배우의 기존 독립 키프레임이 삭제됩니다.

대상 묶기 묶음을 만들면 리더가 아닌 구성원의 기존 독립 키가 삭제됩니다. 묶음을 해제해도 삭제된 키는 자동 복원되지 않습니다.

09

프리뷰와 내보내기

2D 동선도와 3D 카메라 프레임을 확인한 뒤 필요한 파일만 직접 저장합니다.

동선 재생2D·3D 무대에서 움직임과 시간을 검토
내보내기 프리뷰실제 저장될 이미지·MP4를 최종 확인
직접 저장확인한 결과만 사용자가 저장

2D 블로킹 이미지

  • 현재 재생 위치의 배우·카메라 배치와 전체 동선, 키 번호, 방향 화살표를 한 장의 PNG로 준비합니다.
  • AI 입력에서는 3D 가이드 영상과 함께 별도의 `@plan` 공간 참고 이미지로 사용합니다.

현재 프레임

  • 현재 재생 위치의 카메라 프레임을 PNG로 준비합니다.
  • 프리뷰의 `이미지 복사`로 클립보드에 넣어 다른 앱에 붙여넣을 수 있습니다.

시작·끝 프레임

  • 정확히 0초와 전체 시간 끝의 프레임 두 장을 만듭니다.
  • 두 이미지는 확인 후 ZIP으로 저장합니다.

프리비즈 영상

  • 카메라 프레임만 렌더링하며 화살표, 지침, 앵커, 렌즈 정보 같은 화면 표식은 포함하지 않습니다.
  • 출력은 H.264 MP4이며 생성 후 영상 플레이어에서 먼저 재생해 확인합니다.

스토리보드 시트

  • 스토리보드의 `이미지 묶기`에서 4·6·12컷 시트를 준비합니다.
  • 전체 시나리오와 현재 상태 필터가 그대로 적용됩니다.

자동 다운로드 없음 모든 출력은 `내보내기 프리뷰`에서 확인하고 `이 프리뷰로 저장`을 직접 눌러야 파일로 저장됩니다.

같은 대상에 서로 다른 시간의 키가 최소 두 개 있어야 이동이 생깁니다. `대상` 선택과 키 시간을 확인하세요.

전체 시간을 바꿀 수 없어요

잠긴 대상의 키가 있으면 시간 비율 변경으로 키가 움직이지 않도록 전체 시간 입력이 비활성화됩니다. 해당 대상의 편집 잠금을 먼저 해제하세요.

팬·틸트를 직접 바꿀 수 없어요

피사체 추적이 켜져 있거나 카메라 회전이 잠겨 있습니다. 추적 대상을 `추적 안 함`으로 바꾸거나 회전 잠금을 해제하세요.

MP4 준비가 실패해요

FrisFrame 서버로 접속했는지, FFmpeg가 설치되어 있는지 확인하세요. MP4 대신 WebM으로 자동 변경되지는 않습니다.

다른 메뉴 이름이나 작업 용어로 검색해 보세요.
