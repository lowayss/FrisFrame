# FrisFrame

촬영 전 카메라, 배우, 소품, 동선을 2D와 3D에서 함께 설계하는 브라우저 기반 프리비즈 도구입니다.

## 실행

MP4 프리뷰 인코딩을 지원하는 로컬 서버로 실행합니다.

```bash
python3 server.py --port 8766
```

```text
http://127.0.0.1:8766/
```

## 기본 작업 흐름

1. `시나리오·콘티`에 원문과 컷 설명을 입력해 씬·컷 구조를 미리 확인하고 적용합니다.
2. `스토리보드`에서 컷 순서, 액션, 대사, 연출 의도와 검토 상태를 정리합니다.
3. 컷의 `블로킹 열기`에서 배우, 소품, 카메라와 포커스 위치를 2D 또는 3D로 배치합니다.
4. 원하는 시간으로 이동한 뒤 `키프레임 추가`를 눌러 선택한 대상의 키를 만듭니다.
5. 3D 카메라 프레임과 스토리보드 대표 프레임으로 컷의 흐름을 확인합니다.
6. 현재 프레임 PNG, 시작·끝 프레임 ZIP 또는 H.264 MP4 카메라 영상을 준비합니다.
7. 준비 창에서 `파일 저장`을 직접 눌러 저장합니다.

새 작업의 기본 길이는 15초이며 키프레임은 0개입니다. 카메라, 배우, 소품을 움직이거나 추가해도 키가 자동으로 생기지 않습니다. 키는 사용자가 `키프레임 추가`를 눌렀을 때만 생성됩니다. 키가 없는 대상은 전체 시간 동안 현재 위치에 고정됩니다.

## 키프레임

- `대상`에서 카메라, 배우, 소품 또는 전체를 선택합니다.
- `키프레임 추가`는 현재 배치와 시간을 새 키로 저장합니다.
- `선택 키 갱신`은 선택된 키를 현재 배치와 시간으로 바꿉니다.
- 마지막 하나뿐인 키도 삭제할 수 있으며, 키가 없어진 대상은 고정 상태로 돌아갑니다.
- 각 점은 드래그해 시간을 바꿀 수 있습니다.
- `부드럽게`, `일정 속도`, `직전 유지`, `즉시 전환`으로 키 사이 움직임을 정합니다.
- 통합 타임라인과 대상별 타임라인을 전환할 수 있습니다.

타임라인을 문지르거나 재생하고 내보내는 동작은 편집 데이터를 변경하지 않습니다. 화면에서 실제 편집을 시작하거나 키를 추가·갱신할 때만 현재 자세가 반영됩니다.

## 3D와 내보내기

- 3D의 `이동`, `방향`, `시점` 모드로 배우·소품·카메라를 편집하거나 관찰합니다.
- 오른쪽 아래 카메라 프레임은 현재 렌즈와 카메라 높이로 보이는 실제 출력 구도입니다.
- `현재 프레임`은 현재 재생 위치의 카메라 프레임을 PNG로 준비합니다.
- `시작+끝`은 정확히 0초와 전체 길이 끝의 카메라 프레임 두 장을 ZIP으로 준비합니다.
- `카메라 영상`은 전체 시간과 FPS에 맞춰 모든 프레임을 렌더링한 뒤 H.264 MP4로 준비합니다.
- 2D 평면도, 프롬프트, 제작팩은 내보내기 메뉴에 포함하지 않습니다.

MP4 생성에는 로컬 서버와 FFmpeg가 필요하며, 준비에 실패하면 다른 형식으로 바꾸지 않고 오류를 표시합니다.

## 프로젝트와 스토리보드

- 프로젝트는 여러 씬과 컷을 포함하며, 각 컷은 독립된 2D·3D 블로킹과 키프레임을 가집니다.
- 시나리오 헤딩과 `컷`, `액션`, `대사`, `카메라`, `렌즈`, `길이` 표기를 읽어 편집 가능한 초안을 만듭니다.
- 자동으로 찾기 어려운 씬 연결은 적용 전에 미리보기와 경고로 확인합니다.
- 컷 카드는 대표 카메라 프레임, 샷 크기, 렌즈, 길이, 키 개수와 `초안`, `블로킹 중`, `검토 필요`, `확정` 상태를 보여줍니다.
- 기존 단일 작업 JSON은 불러올 때 새 프로젝트의 첫 씬·첫 컷으로 변환됩니다.
- 프로젝트 JSON에는 실행 취소 기록이나 대표 프레임 이미지가 포함되지 않습니다.
- 시나리오·콘티 창을 닫아도 적용 전 입력은 현재 브라우저 세션에 임시로 유지됩니다.
- 씬·컷 추가, 삭제, 복제, 순서 변경과 구조 적용은 스토리보드 화면에서 실행 취소할 수 있습니다.
- 빈 시나리오와 빈 콘티는 기존 프로젝트 구조에 적용할 수 없습니다.
- 스토리보드는 `현재 씬`과 `전체 시나리오` 범위를 전환할 수 있습니다.
- 전체 시나리오에서는 모든 씬과 컷을 씬 순서대로 이어서 스크롤하며 검토합니다.
- 현재 보이는 범위와 상태 필터를 기준으로 `4컷 2×2`, `6컷 3×2`, `12컷 4×3` 이미지 시트를 만들 수 있습니다.
- 시트가 한 장이면 PNG, 선택한 칸 수를 넘어 여러 장이면 PNG들을 ZIP으로 묶어 준비합니다.
- 이미지 시트도 저장 전 프리뷰를 거치며 자동으로 내려받지 않습니다.

## 작업 화면과 단축키

- 상단의 `스토리보드`, `2D`, `3D`에서 작업 화면을 바로 전환합니다.
- `Cmd/Ctrl+1`, `Cmd/Ctrl+2`, `Cmd/Ctrl+3`은 각각 스토리보드, 2D, 3D를 엽니다.
- `Cmd/Ctrl+S`는 프로젝트 파일을 바로 저장하지 않고 저장 프리뷰를 준비합니다.
- 블로킹 화면에서 `Space`는 재생·정지, `K`는 키프레임 추가, `Shift+K`는 선택 키 갱신입니다.
- `[`와 `]`는 이전·다음 키를 선택하고, 스토리보드의 `Alt+←/→`는 선택 컷의 순서를 바꿉니다.
- 입력창을 편집하는 동안에는 캔버스와 타임라인 단축키가 실행되지 않습니다.

## 저장 정책

이미지, 영상, 시작·끝 프레임, 프로젝트는 자동으로 내려받지 않습니다. 내보내기 준비 창에서 파일명과 형식을 확인하고 `파일 저장`을 직접 눌러야 다운로드가 시작됩니다. 코드로 발생시킨 자동 클릭은 무시됩니다.
