# FrisFrame

촬영 전 카메라, 배우, 소품, 동선을 2D와 3D에서 함께 설계하는 브라우저 기반 프리비즈 도구입니다.

현재 개발 범위는 프로젝트, 스토리보드, 수동 블로킹, 키프레임, 3D 카메라 프리뷰와 검토 후 내보내기에 집중합니다. 외부 영상 분석과 외부 3D 에셋 가져오기는 지원하지 않으며, 기본 제공 오브젝트 범위도 당분간 확장하지 않습니다.

## 실행

MP4 프리뷰 인코딩을 지원하는 로컬 서버로 실행합니다.

```bash
python3 server.py --port 8766
```

```text
http://127.0.0.1:8766/
```

업데이트 전후의 전체 안정성 검사는 한 명령으로 실행할 수 있습니다.

```bash
python3 quality_check.py
```

코드 책임, 프로젝트 형식과 Electron 연결 원칙은 [`MAINTENANCE.md`](MAINTENANCE.md)에 정리되어 있습니다.

## macOS 데스크톱 앱

Apple Silicon용 설치 이미지는 `release/FrisFrame-0.2.8-arm64.dmg`입니다. DMG를 열고 FrisFrame을 Applications 폴더로 옮겨 실행합니다. 현재 빌드는 로컬 검증용 미서명 버전이며, 외부 배포 전에는 Apple Developer ID 서명과 notarization이 필요합니다.

기존 로컬 서버 프로젝트를 데스크톱 앱으로 처음 옮기고 패키지를 다시 만드는 명령은 다음과 같습니다.

```bash
npm install
npm run check
npm run desktop:migrate-data
npm run desktop:build
```

프로젝트와 자동 저장 데이터는 앱 파일 밖의 `~/Library/Application Support/FrisFrame/data/frisframe.db`에 보존됩니다. 앱을 교체하거나 업데이트해도 이 DB는 패키징 과정에서 덮어쓰지 않습니다. Three.js와 Lucide는 앱 안에서 오프라인으로 로드하며, MP4용 FFmpeg도 앱 런타임에 포함됩니다.

## 기본 작업 흐름

1. `시나리오·콘티`에 원문과 컷 설명을 입력해 씬·컷 구조를 미리 확인하고 적용합니다.
2. `스토리보드`에서 컷 순서, 액션, 대사, 연출 의도와 검토 상태를 정리합니다.
3. 컷의 `블로킹 열기`에서 배우, 소품, 카메라와 포커스 위치를 2D 또는 3D로 배치합니다.
4. 원하는 시간으로 이동한 뒤 `키프레임 추가`를 눌러 선택한 대상의 키를 만듭니다.
5. 3D 카메라 프레임과 스토리보드 대표 프레임으로 컷의 흐름을 확인합니다.

6. 현재 프레임 PNG, 시작·끝 프레임 ZIP 또는 H.264 MP4 프리비즈 영상을 준비합니다.
7. 준비 창에서 `파일 저장`을 직접 눌러 저장합니다.

2D 무대의 긴 변은 36m로, 이전 24m 기준보다 1.5배 넓습니다. 16:9 무대는 약 36×20.3m이며 100%에서 작업창에 맞춰 표시되고 최대 400%까지 확대됩니다. 확대 상태에서는 빈 무대를 좌클릭 드래그해 이동하며, 가운데 버튼 또는 `Space + 드래그`는 대상 위에서도 무대를 이동합니다.

새 작업은 배우 1명, 소품 0개, 15초, 키프레임 0개로 시작합니다. 카메라, 배우, 소품을 움직이거나 추가해도 키가 자동으로 생기지 않습니다. 키는 사용자가 `키프레임 추가`를 눌렀을 때만 생성됩니다. 키가 없는 대상은 전체 시간 동안 현재 위치에 고정됩니다.

2D 평면도의 동선은 파랑(카메라), 청록(배우), 황색(소품)으로 구분됩니다. 전역 범례 대신 현재 이동 구간 바로 옆에 스토리보드식 화살표와 짧은 지문을 표시합니다. 빈 원은 시작점, 밝은 점은 현재 재생 위치, 십자 표적은 다음 도착점입니다.

## 키프레임

- `대상`에서 카메라, 배우, 소품 또는 전체를 선택합니다.
- `키프레임 추가`는 현재 배치와 시간을 새 키로 저장합니다.
- `선택 키 갱신`은 선택된 키를 현재 배치와 시간으로 바꿉니다.
- 마지막 하나뿐인 키도 삭제할 수 있으며, 키가 없어진 대상은 고정 상태로 돌아갑니다.
- 각 점은 드래그해 시간을 바꿀 수 있습니다.
- `부드럽게`, `일정 속도`, `직전 유지`, `즉시 전환`으로 키 사이 움직임을 정합니다.
- 통합 타임라인과 대상별 타임라인을 전환할 수 있습니다.
- `카메라 높이 키`는 현재 시간의 카메라 키가 있으면 높이만 갱신하고, 없으면 새 카메라 키를 만듭니다.

## 잠금과 연속성

- 카메라는 위치, 회전, 렌즈, 높이를 각각 잠글 수 있습니다.
- 배우와 소품은 `대상 편집 잠금`으로 2D·3D 이동, 속성, 삭제와 키 편집을 함께 막습니다.
- 잠금은 재생 결과를 바꾸지 않으며 기존 키프레임은 그대로 재생됩니다.
- 이어지는 컷은 직전 컷의 마지막 보간 프레임을 시작 배치로 사용합니다.
- 스토리보드의 컷 점검에는 배우 누락, 큰 위치·방향 변화, 탑승 상태와 소품 외형 변화가 연속성 경고로 표시됩니다.

타임라인을 문지르거나 재생하고 내보내는 동작은 편집 데이터를 변경하지 않습니다. 화면에서 실제 편집을 시작하거나 키를 추가·갱신할 때만 현재 자세가 반영됩니다.

## 3D와 내보내기

- 3D에서는 대상 위의 좌클릭과 `G` 이동·`R` 회전으로 대상을 편집합니다. 무대 바닥이나 빈 공간의 좌클릭 드래그와 가운데 버튼은 오빗, 우클릭 또는 `Shift + 가운데 버튼`은 화면 이동, `Ctrl + 가운데 버튼`과 휠은 줌으로 동작합니다.
- 오른쪽 아래 카메라 프레임은 현재 렌즈와 카메라 높이로 보이는 실제 출력 구도입니다.
- 키프레임의 `지문`에는 `웃으며 오른쪽으로 두 걸음`처럼 해당 구간에 표시할 짧은 행동 설명을 기록합니다. 비워 두면 이동 방향에서 기본 지문을 자동 생성합니다.
- `2D 블로킹 이미지`는 현재 재생 위치의 배치와 전체 동선, 키 번호, 방향 화살표, 현재 구간의 지문을 1920px PNG로 준비합니다. 3D 가이드 영상과 함께 별도의 공간 참고 이미지로 첨부할 수 있습니다.
- `현재 프레임`은 현재 재생 위치의 카메라 프레임을 PNG로 준비합니다.
- `시작+끝`은 정확히 0초와 전체 길이 끝의 카메라 프레임 두 장을 ZIP으로 준비합니다.
- `프리비즈 영상`은 전체 시간과 FPS에 맞춰 모든 프레임을 렌더링한 뒤 H.264 MP4로 준비합니다.
- 프리비즈 영상과 카메라 프레임에는 화살표, 지문, 앵커, 렌즈와 카메라 정보 같은 화면 표식을 포함하지 않습니다.
- 2D 평면도, 프롬프트, 제작팩은 내보내기 메뉴에 포함하지 않습니다.

MP4 생성에는 로컬 서버와 FFmpeg가 필요하며, 준비에 실패하면 다른 형식으로 바꾸지 않고 오류를 표시합니다.

## 프로젝트와 스토리보드

- 처음 실행하거나 최근 프로젝트가 없을 때는 프로젝트 관리 화면에서 제목을 입력해 새 프로젝트를 만들거나 기존 프로젝트를 열어야 합니다.
- 새 프로젝트는 제목과 서버 저장이 완료된 뒤 편집 화면으로 열리며, 제목 없는 임시 프로젝트는 저장되지 않습니다.
- 상단 `프로젝트` 메뉴의 `프로젝트 저장`은 현재 작업을 서버의 `내 프로젝트`에 저장합니다. 한 번 저장한 프로젝트는 편집 후 자동 저장됩니다.
- 저장 전 브라우저가 닫히거나 연결이 끊겨도 프로젝트별 복구본을 남기며, 다음 실행에서 서버 저장본과 비교해 복구 여부를 안내합니다.
- 상단 `프로젝트` 메뉴의 `최근 저장본`은 현재 프로젝트의 이전 저장본을 최대 5개까지 보여주며, 현재 상태를 이력에 남긴 뒤 선택한 저장본으로 복원합니다.
- 다음 실행에서는 마지막으로 열었던 관리 프로젝트를 자동으로 이어서 엽니다.
- `프로젝트 관리`에서 최근 프로젝트 검색, 열기, 이름 변경, 복제, 휴지통 이동, 복원과 영구 삭제를 할 수 있습니다.
- 저장본은 서버 revision으로 충돌을 검사하며 다른 창의 변경을 조용히 덮어쓰지 않습니다.
- 삭제는 먼저 휴지통으로 이동하며 휴지통 안에서만 영구 삭제할 수 있습니다.
- JSON은 프로젝트 이동·보관을 위한 별도 백업이며 서버 저장 상태와 구분됩니다.
- 프로젝트는 여러 씬과 컷을 포함하며, 각 컷은 독립된 2D·3D 블로킹과 키프레임을 가집니다.
- 시나리오 헤딩과 `컷`, `액션`, `대사`, `카메라`, `렌즈`, `길이` 표기를 읽어 편집 가능한 초안을 만듭니다.
- 자동으로 찾기 어려운 씬 연결은 적용 전에 미리보기와 경고로 확인합니다.
- 컷 카드는 대표 카메라 프레임, 샷 크기, 렌즈, 길이, 키 개수와 `초안`, `블로킹 중`, `검토 필요`, `확정` 상태를 보여줍니다.
- 기존 단일 작업 JSON은 불러올 때 새 프로젝트의 첫 씬·첫 컷으로 변환됩니다.
- 프로젝트 JSON에는 실행 취소 기록이나 대표 프레임 이미지가 포함되지 않습니다.
- 시나리오·콘티 창을 닫아도 적용 전 입력은 현재 브라우저 세션에 임시로 유지됩니다.
- 씬·컷 추가, 삭제, 복제, 순서 변경과 구조 적용은 스토리보드 화면에서 실행 취소할 수 있습니다.
- 빈 시나리오와 빈 콘티는 기존 프로젝트 구조에 적용할 수 없습니다.
- 스토리보드는 항상 전체 시나리오의 모든 씬과 컷을 씬 순서대로 이어서 표시합니다.
- 왼쪽 씬 목록을 선택하면 전체 시나리오 화면 안에서 해당 씬으로 이동합니다.
- 전체 시나리오와 상태 필터를 기준으로 `4컷 2×2`, `6컷 3×2`, `12컷 4×3` 이미지 시트를 만들 수 있습니다.
- 시트가 한 장이면 PNG, 선택한 칸 수를 넘어 여러 장이면 PNG들을 ZIP으로 묶어 준비합니다.
- 이미지 시트도 저장 전 프리뷰를 거치며 자동으로 내려받지 않습니다.

## 작업 화면과 단축키

- 상단의 `스토리보드`, `2D`, `3D`에서 작업 화면을 바로 전환합니다.
- 상단 `도움말` 메뉴에는 기능별 상세 설명을 제공하는 `기능 매뉴얼`과 화면을 따라가는 `처음 사용 가이드`가 함께 있습니다. 매뉴얼은 `F1`로 열고 닫을 수 있습니다.
- `Cmd/Ctrl+1`, `Cmd/Ctrl+2`, `Cmd/Ctrl+3`은 각각 스토리보드, 2D, 3D를 엽니다.
- `Cmd/Ctrl+S`는 현재 프로젝트를 서버에 저장합니다.
- 블로킹 화면에서 `Space`는 재생·일시정지, `K`는 키프레임 추가, `Shift+K`는 선택 키 갱신입니다. 정지 버튼만 재생 위치를 0초로 되돌립니다.
- `[`와 `]`는 이전·다음 키를 선택하고, 스토리보드의 `Alt+←/→`는 선택 컷의 순서를 바꿉니다.
- 입력창을 편집하는 동안에는 캔버스와 타임라인 단축키가 실행되지 않습니다.

## 저장 정책

이미지, 영상, 시작·끝 프레임과 JSON 백업은 자동으로 내려받지 않습니다. 내보내기 준비 창에서 파일명과 형식을 확인하고 `파일 저장`을 직접 눌러야 다운로드가 시작됩니다. 프로젝트 자동 저장은 서버 데이터베이스에만 기록되며 파일 다운로드를 발생시키지 않습니다. 코드로 발생시킨 자동 클릭은 무시됩니다.

## 외부 배포 보안

외부 주소로 서버를 열 때는 라이선스 인증이 필수입니다. 원본 라이선스 키는 해시로 저장되고 브라우저에는 임의 세션 토큰만 전달됩니다.
외부 배포는 HTTPS를 종료하고 `X-Forwarded-Proto: https`를 전달하는 리버스 프록시 뒤에서 실행해야 합니다. 아래의 pepper는 충분히 긴 임의 문자열로 만들고 서버와 라이선스 추가 명령에 동일하게 전달합니다.

```bash
ENABLE_LICENSE_CHECK=true \
FRISFRAME_REQUIRE_ORIGIN=true \
FRISFRAME_SECURE_COOKIES=true \
FRISFRAME_LICENSE_PEPPER='충분히-긴-임의-비밀값' \
python3 server.py --host 0.0.0.0 --port 8766
```

라이선스는 서버를 실행하기 전에 별도 데이터 경로에 추가합니다.

```bash
PREVIS_DB_PATH=/secure/path/frisframe.db \
FRISFRAME_LICENSE_PEPPER='서버와-같은-비밀값' \
python3 add_license.py --owner "사용자 이름"
```

서버는 앱에 필요한 정적 파일만 허용하며 DB, 서버 코드와 Git 내부 파일을 제공하지 않습니다. 공유 프로젝트는 소유자 세션 또는 링크의 별도 공유 토큰이 있어야 열립니다.

## Model Context Protocol (MCP) 연동

FrisFrame은 AI 모델(예: Claude Desktop, Cursor 등)이 자연어 명령으로 프로젝트 데이터를 직접 제어하고 컷을 구성할 수 있도록 지원하는 표준 MCP 서버를 내장하고 있습니다.

### MCP 서버 실행 및 테스트

별도의 외부 라이브러리 설치 없이 파이썬 표준 라이브러리(Stdio 채널)만을 이용하여 가볍고 안전하게 실행됩니다.

* **동작 검증 (스모크 테스트)**:
  ```bash
  python3 tests/mcp-server-smoke.py
  ```
* **제공하는 AI 도구(Tools)**:
  - `list_projects`: 데이터베이스 내 프로젝트 목록 나열
  - `get_project`: 특정 프로젝트 JSON 문서 로드
  - `create_project`: 신규 프리비즈 프로젝트 생성
  - `save_project`: 수정된 프로젝트 JSON 저장 및 백업 리비전 생성
  - `create_cut`: 지시어 분석 기반 카메라 자동 가배치를 포함하여 스토리보드 컷 추가
  - `update_camera_blocking`: 특정 컷의 3D 카메라 매개변수(위치, 높이, 각도, 화각) 미세 수정
  - `add_actor_to_cut`: 3D 프리비즈 씬에 신규 배우 추가

### Claude Desktop 연동 설정

`~/Library/Application Support/Claude/claude_desktop_config.json` 설정 파일의 `mcpServers` 항목에 아래 내용을 추가합니다.

```json
{
  "mcpServers": {
    "frisframe": {
      "command": "python3",
      "args": [
        "/Users/loways/Documents/영화작업용/stage-blocking-lab/mcp_server.py"
      ],
      "env": {
        "PREVIS_DB_PATH": "/Users/loways/Documents/영화작업용/stage-blocking-lab/previs_projects.db"
      }
    }
  }
}
```

이제 Claude Desktop 등에서 *"FrisFrame 프로젝트 목록 보여줘"*, *"신규 컷 추가하고 카메라는 ELS 하이앵글로 배치해줘"*와 같은 명령어로 프리비즈 씬을 실시간으로 설계할 수 있습니다.
